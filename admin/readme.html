Overview:

  • Downloads plugin adds downloads section to your Geeklog site.
  • This plugin is derived from the famous Filemgmt plugin. And, adds various features.

Features:

  • Create, edit, and delete downloads file information on admin panel.
  • Download file can be classified in multi-level category.
  • Download file can be put tags.
  • The description and details of file can be expressed freely using HTML text.
  • Like the story you can edit using the Advanced Editor.
  • Snapshot thumbnail view.
  • Multi-language support.

Terms:

  • Downloads plugin works in Geeklog 1.8.0 or later. It doesn't work in older.
  • Currently, only MySQL database support.

1) Installation

To install, create some directories for Downloads plugin manually, you must copy some files to the appropriate directory.

In the following descriptions

  • <geeklog_dir> is the directory where the system db-config.php file resides
  • <public_html> is the directory where the lib-common.php file resides
  • <admin> is the directory where the administration files reside (usually, <public_html>/admin/ )

Step 1 - Backup your database

  • Because Downloads plugin adds some database tables, backup your current Geeklog databese beforehand using "DB Backups" feature or phpMyAdmin.

Step 2 - Uncompress the plugin archive

  • Uncompress the Downloads plugin archive in your <geeklog_dir>/plugins directory. The archive will create a directory called downloads in your plugins directory.

Step 3 - Create the required plugin directories

  • Create a directory called downloads under your <public_html> directory.
  • Create a directory called downloads under your <admin>/plugins/ directory.

Step 4 - Copy the required plugin files

  • Change to your <geeklog_dir>/plugins/downloads/ directory.
  • Copy the files and directories in the admin directory to the <admin>/plugins/downloads/ directory your created in step 3.
  • Copy the files and directories in the public_html directory to the <public_html>/downloads/ directory your created in step 3.

Step 5 - Create the required directories for the file repository

  • Create the following directories under <public_html> .
  • downloads_data
  • downloads_data/category_snaps
  • downloads_data/files
  • downloads_data/snaps
  • downloads_data/tn
  • You might also copy your <geeklog_dir>/plugins/downloads/downloads_data under <public_html>.
  • Set directory permission so that your webserver has write access to the directories. This may require write permissions to be 744, 754 or 777 depending on your webserver setup.

Step 6 - Run the plugin install from the plugin editor

  • Log in to your Geeklog as a root user and run the Downloads plugin install from the plugin editor ("Plugins").
  • If the install was not successful, examine Geeklog system errorlog (error.log).
  • Downloads plugin was installed now.
  • Clicking on the "Downloads" will take you to the "Downloads Manager" page.

2) Uninstallation

Step 1 - Backup your database

  • During the uninstall process, Downloads plugin table will be deleted, and we recommend that you backup your current Geeklog databese beforehand using "DB Backups" feature or phpMyAdmin.

Step 2 - Run the plugin uninstall from the plugin editor

  • Run the Downloads plugin uninstall from the plugin editor ("Plugins"). This will remove Downloads plugin table and all data from your database.

Step 3 - Delete directories

  • Delete the three plugin directories created in the install process ( <geeklog-dir>/plugins/downloads/, <public_html>/downloads/, <admin>/plugins/downloads/ ).

Step 4 - Delete directories for the file repository

  • Delete the directories created in 1) - Step5.

4) Try out

Let's use Downloads plugin.

Step 1 - Show Downloads manager

  • Click the "Downloads" from the Admin Menu.

Step 2 - Add a category

  • You must create at least one or more categories.
  • Click "Add Category".

  • On "Add category" screen, enter "Category ID" and "Title", and then click "Save Changes" button.

  • One category have been added.

Step 3 - Add a download file

  • Download the file (information) to add.
  • Click "Add File".

  • Enter the minimum "Title", "File ID", "File Name", "Category", "Description", and then click "Add" button.
  • "Category" is already selected categories you created in Step 2.

  • One download file have been added.

Step 4 - See downloads section

  • Click the "Downloads" from the header menu.
  • See the download section. This is configured in the category list and the file list.

5) Managing categories

Hereunder are items to be edited at category manager.

Item Desctription
Category ID Set category ID here. Alphabet, numerics, hyphen, under score are allowed. ID duplication is not allowed.
If your Geeklog site is operated by multi-language mode, language ID should be added on category ID. (ex:cat_en, cat_ja ...)
Title Set category title here.
Category Image Set category icon's file name here.
You can upload new category icon image file by clicking 'Upload' button.
Parent Category Select upper layer category.
Order Set display order at category list here. Value here is to be adjusted to decimal order by Geeklog.
Enabled Switch each category's status enabled/disabled.
If disabled, files & categories belong to do not display at downloads section of your Geeklog site.

6) Managing downloads

Hereunder are items to be edited at downloads manager.

Item Desctription
Submitter Who submitted the file is automatically set here.
Title Set the file's title here.
File ID Set the file's ID here. Alphabet, numerics, hyphen, under score are allowed. ID duplication is not allowed.
File Name Set the file's name here.
Replacing file is available by click 'Upload'.
Category Choose a category you wish the file belongs to.
Tag Set the tag list separated by a space character.
In order to enable the tag, install the Tag plugin(Ver 0.6.0 or later) that is released by mystral-kk.
Web Describe URLs such as the one of the submitter's web.
Geeklog provides user's web URL.
Version Set the file's version here.
File Size The file's size is described automatically here.
MD5 Checksum The file's MD5 value is described automatically here.
Description Describe the file's description here. Simple one is recommended.
HTML is acceptable when post mode is [HTML] or[advanced editor].
Detail Describe details of the file.
HTML is acceptable when post mode is [HTML] or [advanced editor].
Post Mode Chosse post mode of description and details.
[text] : plain text
[HTML] : HTML text
[advanced editor] : edit HTML text by advanced editor
[Wiki text] : Wiki text
File Image Set file name of snapshot image.
New file upload is available by clicking 'Upload' button.
MG Autotag Set Autotag of Media Gallery.
Once MG Autotags is set, a file image (snapshot) is replaced by the one created by MG Autotags.
This item appears when 'MG Autotag' is enabled at Geeklog's configuration.
Comment Mode Choose comment actions
[Comments Enabled] : allow comment submit
[Comments Disabled] : disallow comment submit
[Comment Closed] : close comment features
Released Select if the file & its information make it in public.
[YES] : make it in public (accessable)
[NO] : not to make it in public (access unavailable)
Include Files on List Select if the file's download information is displayed on the list, ie numbers of downloads etc.
[YES] : download information not displayed
[NO] : download information is displayed
This selection is not to be effected by the previous item 'Released'.
Release Date Set download information's release date
If future date is set, the file does not to be in public before the date.
Project Name Descript project name of the download file.

8) Multi-language support

Downloads Plugin supports Geeklog's multi-language features.

Downloads plugin uses a method to add language ID at suffix of category ID.

For details of multi-language support, please see the following WIKI pages.

  • http://wiki.geeklog.jp/index.php/Multi-Language_Support#.E3.83.96.E3.83.AD.E3.83.83.E3.82.AF (Japanese)
  • http://wiki.geeklog.net/index.php/Multilingual_Support (English)

The following is an example to use at English and Japanese.

  • Create categories for each language by category editor.
  • Set 'category ID' like below.
plugin_en
plugin_ja
  • The one suffix of "_en" is used for English mode, suffix "_ja" for Japanese mode.
  • Creat 2 download information for each language's category. In this case, for English and Japanese.
  • This is all what you have to do. Try and see switching launguage at user's setting.
